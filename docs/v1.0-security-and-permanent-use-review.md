# v1.0 Security and Permanent-Use Review

Review date: 2026-05-13

## Verdict

BLUE-TANUKI has no known final-review bypass, hidden authority source, or
undocumented privileged operation in the reviewed v1.0-prep surface.

This review does not claim that every preview channel is first-party release
quality. Slack, Discord, Teams, LINE, and browser automation remain preview or
preview-quarantined where documented. WhatsApp remains `reserved-third-party`.

## Boundary Checks

### Final-Review Boundary

`FINAL_REVIEW_OPERATIONS` includes:

- `tool.file.delete`
- `tool.shell.exec`
- `external.send`
- `credential.access`
- `settings.write`
- `schedule.create`
- `schedule.update`
- `schedule.delete`
- `browser.automation`
- `github.write`
- `google.write`
- `payment.charge`

`evaluateApproval()` still returns `ask` for these operations even when a
reusable grant matches. `full_access` only auto-allows non-final-review work.

### Runtime Invariants

Expected values remain:

```json
{
  "hds_calls_llm": false,
  "process_policy_enforced": true,
  "external_metadata_can_escalate_authority": false,
  "memory_used_for_authority": false,
  "final_review_boundary_enforced_by_approval_gate": true
}
```

### Preview and Support Boundary

- Slack / Discord: release-polished preview, not first-party promotion.
- Teams / LINE: first-party-preview, owner credentialed live smoke still required.
- Browser automation: disabled-by-default preview.
- WhatsApp: reserved-third-party, not first-party core.
- Signed native installer: not shipped.
- Automatic updater: not shipped.

## Open Findings

No Phase 11-S1 release-blocking security finding was identified.

## Residual Risks

- Cross-platform installer e2e was not executed on Windows, macOS, and Linux in
  this review; coverage is static docs/script validation plus release bundle
  checks.
- Credentialed live smoke was skipped where external tokens or targets were not
  configured.
- Preview channels require owner-run live credentials and recovery review
  before first-party promotion.
- Browser automation remains preview because credentialed sessions, uploads,
  downloads, and side-effecting actions are intentionally unsupported.
