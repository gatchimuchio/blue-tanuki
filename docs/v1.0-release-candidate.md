# BLUE-TANUKI v1.0 Release Candidate

Release candidate: `1.0.0-rc.1`

## Claim

BLUE-TANUKI v1.0 RC is a local, owner-operated resident AI control plane where
HDS-BRAIN owns authority and all LLMs, tools, channels, memory, cron, browser
automation, UI, external APIs, and distribution surfaces remain downstream
devices.

The release claim is limited to BLUE-TANUKI's authority path and permanent-use
operation surface. It is not a claim that external platforms, operating systems,
LLM providers, browser engines, or user-provided tools are internally
transparent.

## Supported Surface

- WebChat Control Center and local HTTP surfaces
- HDS-BRAIN authority path and Runtime Invariants
- Approval Gate with non-bypassable final-review operations
- Hash-chain audit dump and verification
- Runtime schedule list/create/update/delete with L3 final-review for mutation
- Resident notifications as display-only status
- Telegram as first-party channel
- GitHub read/write downstream tools with host/repo/token boundaries
- Google read/write downstream tools with host/token/final-review boundaries
- File, HTTP, browser-read, browser-preview, and shell tools with documented guards
- Portable source bundle, installers, uninstallers, update/rollback runbook, and distribution readiness gate

## Preview / No-Support Boundary

- Slack, Discord, Teams, and LINE remain `first-party-preview`.
- Browser automation remains disabled-by-default preview.
- WhatsApp remains `reserved-third-party`; no first-party WhatsApp core support is shipped.
- Voice, mobile, rich canvas, public third-party Skill registry, and commercial SaaS framing are not part of this release candidate.
- No signed native installer is shipped.
- No automatic updater is shipped.

## Version and Bundle

- Workspace version: `1.0.0-rc.1`
- Release artifact name: `blue-tanuki-1.0.0-rc.1-source-bundle`
- Bundle integrity: use the generated `.sha256` and `.manifest.json` sidecars.
- No hard-coded archive SHA is recorded inside this document because this document is included in the source bundle, so changing the embedded SHA would change the archive SHA again.
- Bundle type: unsigned source bundle
- Bundle excludes `node_modules`, `.blue-tanuki`, local env files, backup env files, private keys, audit/session data, and release output.

## Upgrade Notes

1. Stop the gateway before replacing source or bundle files.
2. Back up the env file, env `.bak` files, audit directory, session directory, memory directory, schedules directory, and approval grants file.
3. Verify the current audit chain before update.
4. Install/update dependencies and rebuild.
5. Run `doctor` before start.
6. Reuse the previous env file and retained local data unless intentionally resetting config.

See [UPDATE_ROLLBACK_RUNBOOK.md](UPDATE_ROLLBACK_RUNBOOK.md) for source and bundle rollback steps.

## First-Run Proof

The first-run claim remains a guided local operation path, not a verified
5-minute beginner guarantee. Required proof:

- `pnpm install --frozen-lockfile`
- `pnpm typecheck`
- `pnpm build`
- `pnpm test`
- `pnpm run doctor`
- WebChat starts with separated `WEBCHAT_TOKEN` and `WEBCHAT_RESUME_TOKEN`
- `/runtime/snapshot` reports expected Runtime Invariants
- final-review operation reaches pending approval before execution

## Permanent-Use Proof

Permanent-use readiness requires:

- startup path selected and documented
- env/audit/session/memory/schedules/approval storage locations known
- backup and restore path tested or rehearsed
- audit verification passing
- `distribution_readiness` doctor check passing
- update/rollback runbook understood
- preview channel status understood before enabling external credentials
